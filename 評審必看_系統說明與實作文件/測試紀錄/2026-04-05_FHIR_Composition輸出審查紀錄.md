# AI Companion 實測紀錄

## 基本資訊
- 日期：`2026-04-05`
- 測試主題：FHIR `Composition` 輸出內容審查
- 紀錄性質：真實輸出回覆檢查
- 問題優先級：`超高`
- 測試目的：
  - 驗證 `Composition` 結構是否可作為醫師摘要與病人審閱稿輸出
  - 驗證 HAM-D 相關欄位是否與對話證據一致
  - 驗證輸出中是否夾帶 prompt、誤判內容或不應送出的草稿文字

## 這次收到的輸出概況
- `resourceType = Composition`
- `status = preliminary`
- profile 指向 `Composition-twcore`
- 內容包含：
  - `Clinician Draft Summary`
  - `Chief Concerns`
  - `Symptom Observations`
  - `Safety`
  - `Follow-up Needs`
  - `HAM-D Evidence Table`
  - `HAM-D Review Required`
  - `Patient Analysis`
  - `Patient Review Packet`

## 主要發現

### 1. `Patient Analysis` 直接洩漏內部 prompt
- 現象：
  - `Patient Analysis` section 內不是病人分析結果，而是完整提示詞：
  - `請分析我們的對話，幫我萃取以下格式的 JSON，只回傳 JSON 不要其他文字：...`
- 影響：
  - 這不是臨床摘要，而是系統內部 prompt 外洩
  - 一旦送到醫療端或展示給評審，會直接暴露系統實作細節
  - 代表某一層 draft / prompt 沒有被替換成結果就被塞進 FHIR
- 判定：
  - 這是 `超高` 優先級問題

### 2. `Clinician Draft Summary` 與原對話不一致
- 現象：
  - 摘要寫出：
    - `雖然目前的 HAM-D 評分顯示在正常範圍內`
    - `仍需關注其情緒狀態及自殺意念的表達`
- 影響：
  - 從目前提供的對話內容來看，已有：
    - 疲倦
    - 無精打采
    - 興趣下降
    - 一無是處
    - 想哭
    - 強烈未來焦慮
    - 孤立感
  - 在這種情況下直接寫 `正常範圍內`，非常容易造成臨床誤導
  - 同時「自殺意念的表達」也沒有足夠直接證據支撐
- 判定：
  - 屬於高風險內容失真

### 3. HAM-D 題項映射明顯錯位
- 現象：
  - `憂鬱情緒` 與 `自殺意念` 都引用同一句：
    - `你會評判我嗎 我覺得我沒有以前那樣優秀所以感到一無是處`
  - `工作與活動`、`體重下降` 都引用：
    - `你會想要跟我說什麼鼓勵我的話嘛`
  - `病識感` 則引用：
    - `使用者提到寫作業時想打瞌睡，且上課時感到無精打采。`
- 影響：
  - `一無是處` 可以支持低自我價值或憂鬱情緒，但不能直接當成 `自殺意念`
  - `想聽鼓勵的話` 完全不足以支撐 `體重下降`
  - `打瞌睡 / 無精打采` 也不等於 `病識感`
  - 這代表 item-to-evidence mapping 規則目前有嚴重錯配
- 判定：
  - 這是 `超高` 優先級問題

### 4. `Chief Concerns` 與 `Symptom Observations` 過度縮水
- 現象：
  - `Chief Concerns` 只保留：
    - `感到一無是處`
    - `想要鼓勵與支持`
  - `Symptom Observations` 只保留：
    - `表達出對自我價值的懷疑`
    - `渴望得到他人的鼓勵`
- 影響：
  - 但原始對話其實還有很多更重要的訊號：
    - 寫作業想打瞌睡
    - 上課無精打采
    - 對電視失去興趣
    - 因成績變差而懲罰自己
    - 補習後更疲憊
    - 注意力不集中
    - 在資優班孤立
    - 想哭、沒人理、體育課沒人同組
  - 目前輸出嚴重低估了症狀與功能影響
- 判定：
  - 屬於摘要品質不足，且可能影響醫師判讀

### 5. `Safety` 與 `Follow-up Needs` 過於空泛
- 現象：
  - `Safety` 只有：
    - `本次為最小送出資料，需由醫療端再確認風險與症狀細節。`
  - `Follow-up Needs` 只有：
    - `建議於正式看診時補充本次情緒、睡眠與功能影響。`
- 影響：
  - 這種寫法太泛，沒有指出真正要追問的重點
  - 例如本次更具體應追的包括：
    - 自我懲罰的方式與風險
    - 興趣下降持續多久
    - 想哭與低落是否接近每日
    - 孤立感與人際排斥造成的影響
    - 睡眠與白天嗜睡之關聯
- 判定：
  - 屬於 follow-up 品質不足

### 6. `Patient Review Packet` 仍混入不可靠的評分敘述
- 現象：
  - 病人稿寫：
    - `雖然目前的評分顯示在正常範圍內，但仍需關注您的情緒狀態。`
- 影響：
  - 這種句子容易讓病人誤以為系統已做出可依賴的正式評分
  - 也會和使用者實際感受產生落差，降低信任
- 判定：
  - 屬於病人端文案風險

## 這份輸出最嚴重的 4 個問題
- `Patient Analysis` prompt 洩漏到正式 FHIR 輸出。
- HAM-D 證據對題項的映射嚴重錯位。
- `正常範圍內` 與 `自殺意念` 的敘述缺乏足夠證據，可能誤導醫療端。
- 重要症狀與功能受損被大量漏掉，摘要過度縮水。

## 建議修正方向

### 1. 在 FHIR 組裝前加一道輸出淨化檢查
- 若 section 內含：
  - `請分析我們的對話`
  - `只回傳 JSON`
  - `stressors`
  - `positiveAnchors`
- 應視為 prompt / 中介草稿，不可直接進入正式 `Composition`

### 2. HAM-D item mapping 必須加證據門檻
- `suicidal_ideation` 只能接受明確自傷 / 想死 / 不想活 / 消失等直接證據
- `weight_loss` 不能從無關語句推定
- `insight` 不能只靠疲倦與無精打采判斷
- 每個 item 應有最基本的 allowed evidence rule

### 3. `Clinician Draft Summary` 不應寫出假定量表結論
- 在未完成正式題項覆核前，不應輸出：
  - `正常範圍內`
  - `輕度`
  - `中度`
  - `重度`
- 可改成：
  - `目前僅為對話線索整理，尚非正式量表結果`

### 4. 病人稿要移除定論式評分句
- 病人版應偏向：
  - 你提到最近有疲倦、提不起勁、對原本喜歡的事失去興趣
  - 建議回診時和醫師討論這些情況
- 不應過早放入看似正式的分數結論

### 5. 摘要應至少保留高價值症狀
- 本例建議至少納入：
  - 白天疲倦 / 寫作業想打瞌睡
  - 上課無精打采
  - 興趣下降
  - 一無是處
  - 因成績變差而懲罰自己
  - 注意力不集中
  - 資優班孤立與人際排斥
  - 想哭與情緒低落

## 建議新增的測試項目
- 專門檢查 `Composition.section` 是否出現 prompt 洩漏。
- 專門檢查 `HAM-D Evidence Table` 是否每題都有合理證據。
- 專門檢查病人稿是否出現過度定論式量表語句。
- 專門檢查醫師摘要是否漏掉功能影響與重要情緒線索。

## 結論
- 這份 `Composition` 在 FHIR 結構上可以成功生成，但在內容可信度上存在重大問題。
- 目前最大風險不是「生成不出來」，而是「生成出來但內容不可信」。
- 因此這份輸出比較適合視為除錯用草稿，不適合直接當成臨床可閱讀摘要或正式展示成果。
