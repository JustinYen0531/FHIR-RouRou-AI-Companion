# AI陪伴系統 30 分鐘介紹影片：完整執行方案

這份文件為 30 分鐘介紹影片提供完整的「逐字稿」、「操作流程」與「測試清單」。你可以直接依照此文件進行預演與拍攝。

---

## 第一個部分：拍攝前環境測試清單 (Checklist)

在開始錄影前，請務必確認以下功能正常運作：

- [ ] **伺服器啟動**：執行 `node app\fhirDeliveryServer.js` 後，訪問 `http://localhost:8787/` 正常。
- [ ] **API 密鑰**：在系統的 `Settings` 中已填入正確的 Google 或 Groq API Key（或已設為環境變數）。
- [ ] **健康檢查**：訪問 `http://localhost:8787/health` 顯示 `ok: true` 且 `fhir_server_url` 正確（建議指向 `https://hapi.fhir.org/baseR4` 進行 Demo）。
- [ ] **記憶功能**：隨便聊兩句，確認 `Chat` 頁面右上角的心理驚嘆號（Memory Badge）會增加，且點開能看到記錄。
- [ ] **報表生成**：輸入「幫我整理給醫生」，確認 `Reports` 頁面能產生內容。
- [ ] **FHIR 上傳**：點擊「數位授權並送出報告」，確認能滑動到底部並點擊「同意送出」，且最終收到成功的 JSON 回傳。

  >

  >
  > 我們的系統定位在『診前整理』。它不只是個聊天機器人，它結合了 **AI 陪伴互動** 與 **醫療標準化數據架構（FHIR）**。

  > 這 30 分鐘我會帶大家從環境啟動、實際對話演練、AI 如何形成長期記憶、到最後如何把對話整理成符合 FHIR 標準的格式，並真正送入醫療端伺服器。」

### 06:00 - 15:00 | 段落三：陪伴式對話演示 (核心重點)
**【畫面：進到 Chat 介面，準備開始對話】**
- **口述**：
  > 「現在我們進入對話。我們先從最自然、低壓力的對話開始。注意到右下角的模式，系統預設是『自動分流』，它會自動判斷使用者的情緒狀態。」
- **操作 (請準備複製貼上)**：
  1. **輸入**：`最近覺得好累，什麼事都不想做，晚上也翻來翻去睡不著。`
  2. **等待回覆**：觀察 AI 如何接住情緒（Soulmate 模式）。
  3. **輸入**：`這種狀況大概維持兩週了。我開始懷疑是不是工作壓力太大的關係，但我不敢跟同事說。`
- **口述**：
  > 「大家有沒有注意到，右上角那個心理圖標出現了數字？這就是系統的『認識你（Therapeutic Memory）』模組在運作。它會自動從對話中提取：主要壓力來源、症狀觀察與溝通偏好。
  >
  > 點開這個 Memory Drawer，我們可以看到系統已經記住了我的睡眠問題與工作壓力。這就是為什麼下一次聊天時，它不會再問重複的問題，而是能建立長期的陪伴感。」

### 15:00 - 22:00 | 段落四：多維度報告生成

- **口述**：
  > 「對話結束後，重頭戲來了。我們不需要人工寫摘要。只要輸入『幫我整理給醫生』，系統會啟動多維度輸出引擎。
  >
  > 我們切換到報告頁面。這裡分為：
  > 1. **醫師摘要**：這是給臨床端看的，它會自動對應 HAM-D 等線索，列出核心困擾。
  > 2. **病人分析**：這是用友善、白話的語言，幫病人整理出他最近的心理畫像，這有助於覺察。
  > 3. **FHIR 草稿**：這點最關鍵。我們可以看到這不再是文字，而是符合 TW Core 導向的 JSON 格式。」

## 第四個部分：錄製建議與注意事項

1. **網路穩定性**：因為會呼叫 Google/Groq API 以及 HAPI FHIR，請確保錄影環境網路流暢。
2. **重置數據**：若錄壞了，可以去 `Settings` 或點擊 Memory Drawer 裡的「清除記憶」，將畫像重新歸零。
3. **分段錄影**：30 分鐘很長，建議分為「啟動與開場」、「對話過程」、「報表與 FHIR」三個大段落分開錄製，最後再剪輯在一起。
4. **停留時間**：在展示 JSON 或 醫師摘要時，請停留 3-5 秒，方便觀眾閱讀或你後續加字龍。
