# Micro-intervention Engine V1

## 為什麼需要微介入
目前系統已能做觀察、記錄、整理，但在「使用者有點累、有點亂、還沒到危機」的區間，還缺一層很輕的陪伴動作。微介入卡片的目的不是治療，也不是糾正使用者，而是在對方還有一點互動空間時，給一個幾乎不費力的小入口。

它的設計原則只有三個：
- 陪伴感優先，不像老師發指令
- 一次只做一件很小的事
- 一旦進入高風險或安全分流，就完全讓位給 safety 流程

## 初版卡片清單
### 1. 平靜時刻
- 用途：焦躁、心很亂、需要先穩住一下
- 時長：30 秒
- CTA：`開啟引導呼吸`

### 2. 喝一口水
- 用途：低能量、短句、卡住、不想動
- 時長：10 秒
- CTA：`看看這張小提醒`

### 3. 站起來伸展
- 用途：煩躁、緊繃、肩頸卡住、坐不住
- 時長：20 秒
- CTA：`開啟伸展引導`

### 4. 先不用想很多
- 用途：腦中很亂、不知道先做什麼、互動負擔高
- 時長：15 秒
- CTA：`看看兩個小選項`

## 觸發矩陣
| 訊號 | 優先卡片 | 說明 |
|---|---|---|
| `burden_level_state.burden_level = high` | 喝一口水 / 先不用想很多 | 先給最低負擔行動 |
| `response_style = option_first` | 先不用想很多 | 直接提供極簡選擇 |
| 疲累、沒力、不想動 | 喝一口水 | 最短、最不費力 |
| 焦躁、緊張、喘不過氣 | 平靜時刻 | 先回到呼吸 |
| 煩躁、卡住、肩頸緊 | 站起來伸展 | 讓身體先鬆一點 |
| 不知道怎麼辦、腦袋很滿 | 先不用想很多 | 降低選擇壓力 |

## 不觸發的情境
- `route = safety`
- `risk_flag = true`
- `warning_tags` 出現自傷/自殺高風險訊號
- 使用者正在要求 `FHIR Draft`、醫師摘要或其他重型輸出
- 當前畫面不是聊天頁
- 同類卡片仍在冷卻中
- 使用者剛連續兩次關閉或選擇「先不要」

## UI Flow
1. 使用者送出訊息
2. AI 回覆完成
3. 前端根據 `metadata + session_export + chat history` 判斷是否出卡
4. 若符合條件，只顯示 1 張卡片，不寫入聊天紀錄
5. 點卡片後進入全螢幕引導頁，內容來自 `docs/micro_interventions/*.md`
6. 使用者可選 `我做完了` 或 `稍後再說`
7. 關閉後進入冷卻或 snooze，避免過度打擾

## 文案原則
- 使用繁中陪伴語氣
- 不下命令，不評分
- 不要求使用者證明自己做了
- 盡量用「如果你願意」「我們先」「只做一點點」這種低壓語氣

## 後續可擴充方向
- 依 `TherapeuticMemory.positiveAnchors` 個人化卡片文案
- 依使用者互動結果調整卡片排序
- 增加睡前安定、短暫走動、感官定位等卡片
- 視需要把卡片互動紀錄寫入本地暫存分析，但不直接進 FHIR
