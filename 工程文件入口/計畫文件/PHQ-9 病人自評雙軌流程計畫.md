# PHQ-9 病人自評雙軌流程計畫

## 摘要
這個功能會把 PHQ-9 做成一個和聊天平行、但不互相打架的自評入口。病人在進入對話後，可從右下角藍色圓形按鈕進入 PHQ-9 專頁，逐題填答 9 題分數，並可在每題下方補充正式敘述。完成後，系統會把最新一版 PHQ-9 結果連同補充敘述，持續帶入後續對話提示詞，讓 AI 同時掌握自然對話脈絡與量表趨勢，形成雙軌醫療流程。

## 主要變更
1. 新增 PHQ-9 獨立介面
   - 在聊天頁右下角加入藍色圓形浮動按鈕。
   - 點擊後切換到專屬 PHQ-9 頁面，只顯示量表，不混入一般聊天內容。
   - 頁面提供返回聊天的導覽，並保留未送出的草稿。

2. 建立 PHQ-9 資料結構與儲存
   - 每份 PHQ-9 會保存 9 題分數、每題自由敘述、總分、完成時間與版本資訊。
   - 保留歷次填寫紀錄，並標記最新一筆作為後續對話使用的主版本。
   - 讓同一段對話中的再次填寫，可以覆蓋提示詞中的最新量表資訊，同時保留趨勢軌跡。

3. 把 PHQ-9 注入提示詞與對話狀態
   - 後續每次聊天送出時，將最新 PHQ-9 分數與補充敘述一併放進模型上下文。
   - 注入位置會和既有 therapeutic memory 分開，避免彼此污染。
   - 若有新一版 PHQ-9，系統會優先使用最新結果，舊版保留給歷史與趨勢判讀。

4. FHIR 與後端同步
   - 將 PHQ-9 對應到 QuestionnaireResponse，並在需要時補成可輸出的觀察摘要。
   - 讓 FHIR draft 與 session export 可以讀到 PHQ-9 的最新資料。
   - 保持現有病人記憶、HAM-D、摘要輸出流程不被破壞。

5. UI 與互動規則
   - 每題提供清楚的 0 到 3 分選項。
   - 每題下方加入自由敘述欄位，讓病人補充正式說明。
   - 整體設計偏向低壓、可中斷、可回來續填，避免量表體驗像考試。

## 驗收與測試
- 從聊天頁按浮動按鈕可以穩定進入 PHQ-9 頁面。
- PHQ-9 可逐題填分，並能保存每題補充敘述。
- 送出後，下一輪聊天 request 會帶入最新 PHQ-9 摘要。
- 再次填寫 PHQ-9 後，系統會用新版本覆蓋提示詞中的最新內容。
- FHIR / session export 可看到對應的 QuestionnaireResponse 或摘要資料。
- 既有聊天、記憶、HAM-D、FHIR 草稿流程仍可正常運作。

## 預設假設
- PHQ-9 是獨立於現有 HAM-D 的自評模組，不取代原本的對話式量表追問。
- 預設以最新一筆 PHQ-9 作為 prompt 注入來源，歷史版本只用於趨勢與紀錄。
- 每題自由敘述為選填，但若有填寫，會完整保留並帶入後續對話。
- 不會在這一版直接做醫療診斷或自動結論，系統只負責整理與傳遞資訊。

## 實作備註
- 前端已加入獨立 PHQ-9 畫面與右下角浮動按鈕。
- 對話送出時會一併帶入最新 PHQ-9 context。
- 後端 session export 與 FHIR bundle 已預留 PHQ-9 資料欄位。
