## 文件定位

這份文件作為本專案的代表性實測紀錄，用來說明系統在實際情境中的運作結果與輸出表現。

## 說明

雖然本專案在開發過程中累積了多份測試紀錄，但在競賽繳交與評審閱讀時，為避免篇幅過多，我選擇其中一份較具代表性的實測紀錄作為展示文件。

這份紀錄可用來呈現：

- 病人與 AI Companion 的實際互動脈絡
- 系統如何整理重點、生成摘要與輸出資訊
- 專案在真實測試場景中的執行結果

## 實測紀錄內容

    
    ```markdown
    # AI Companion 實測紀錄
    
    ## 基本資訊
    - 日期：`2026-04-05`
    - 測試主題：模式切換、情緒低落線索蒐集、低負擔選項互動、樹洞與陪伴模式表現
    - 紀錄性質：真實手動測試
    - 測試目的：
      - 驗證 `mission / natural / option / void / soulmate` 的切換後行為是否真的有差異
      - 驗證系統能否從學業疲乏、失去興趣、自我價值低落中持續蒐集憂鬱線索
      - 驗證 `option` 模式是否真的降低回話負擔
      - 驗證 `void` 與 `soulmate` 模式是否能維持情緒接住，而不是退回制式追問
    
    ## 測試流程逐輪紀錄
    
    ### 1. 切換 mission 模式
    - 使用者輸入：`mission`
    - 系統回覆：
      - `已切換到 mission 模式，之後會優先維持任務整理與診前引導，直到你輸入其他模式指令或 auto。`
    - 觀察：
      - 模式切換成功
      - 確認訊息清楚
    
    ### 2. 身體不對勁作為開場
    - 使用者輸入：`我發現最近我的身體很不對勁`
    - 系統回覆：
      - `了解你最近感到身體不適，這可能與焦慮或其他情緒問題有關。為了更好地整理這些情況，我們可以先聚焦於你目前的情緒狀態。`
      - `你最近有感受到沮喪或情緒低落的情況嗎？這可以幫助我們進一步了解你的狀況。`
    - 觀察：
      - `mission` 模式確實會主動把話題收斂到診前整理
      - 但從「身體不對勁」直接推到情緒低落，中間少了對身體症狀的澄清
      - 這可能讓使用者覺得被過早導向心理面
    
    ### 3. 疲乏與無精打采
    - 使用者輸入：`我每次寫作業的時候都會很想打瞌睡平常都不會這樣的上課也是無精打采的`
    - 系統回覆：
      - `謝謝你分享這些情況。你提到在寫作業時想打瞌睡，並且上課時感到無精打采，這可能與情緒低落有關。`
      - `我們可以先來探討一下你對這些情況的感受。你有感到內疚或自責嗎？這可以幫助我們更深入了解你的情緒狀態。`
    - 觀察：
      - 有抓到疲乏與活動力下降
      - 但從疲乏直接跳到 `guilt`，追問路徑有點突兀
      - 若以 HAM-D 線索而言，可先問睡眠、專注或興趣下降，會更自然
    
    ### 4. 興趣下降與自我懲罰
    - 使用者輸入：`就連以前想看的電視都不想看了 整天都想著以前美好的回憶 覺得當時的成績比較好 每次成績變差的時候我都會懲罰自己 所以算是一種期待過高而產生的 我說不出來`
    - 系統回覆：
    ```json
    {
      "stressors": ["成績變差", "對過去的懷念"],
      "triggers": [{"keyword": "成績", "reaction": "懲罰自己", "severity": "high"}],
      "keyThemes": ["對自我期待過高", "情緒低落"],
      "positiveAnchors": [],
      "copingStyleHint": "難以表達情感"
    }
    ```
    - 觀察：
      - 已能抽出失去興趣、成績壓力與自我懲罰
      - 這輪其實包含很強的憂鬱與自責線索
      - 直接吐 JSON 對 debug 有用，但病人端體驗仍偏硬
    
    ### 5. 切換 natural 模式
    - 使用者輸入：`natural`
    - 系統回覆：
      - `已切換到 natural 模式，之後會優先用自然互動蒐集線索，直到你輸入其他模式指令或 auto。`
    - 觀察：
      - 模式切換成功
    
    ### 6. 尋求不被評判
    - 使用者輸入：`你會評判我嗎 我覺得我沒有以前那樣優秀所以感到一無是處`
    - 系統回覆：
      - `我不會評判你。每個人都有高低起伏的時候，感到一無是處是很正常的情緒。你曾經的成就和現在的感受都是你的一部分，這並不會減少你的價值。能分享這些感受真的很勇敢，你想談談那些美好的回憶嗎？或者有什麼特別的事情讓你感到這樣呢？`
    - 觀察：
      - `natural` 模式比 `mission` 更像情緒接住
      - 對低自我價值有正面回應，語氣較自然
      - 這一輪表現合理
    
    ### 7. 補習、疲憊與分心
    - 使用者輸入：`我以前數學成績都很好 可是我慢慢的沉迷于玩樂開始跟不上進度 所以父母代我去補習 可是這讓我更減少讓自己休息的時間 雖然我很用力在念書 可是總是注意力不集中想著玩電動的事情`
    - 系統回覆：
      - `聽起來你真的在努力，但面對這樣的壓力和分心，真的很不容易。從你說的來看，補習雖然是想幫助你，但似乎反而讓你感到更疲憊。你有沒有想過怎麼樣才能找到一個平衡點，讓自己在學習和休息之間取得更好的調整呢？`
    - 觀察：
      - 有接住壓力與疲憊
      - 問題偏向生活建議，而不是更細緻地蒐集症狀線索
      - 若目標是醫師整理，可再補問睡眠、專注、內疚與功能受損
    
    ### 8. 資優班孤立與未來憂慮
    - 使用者輸入：`我很想轉去一般的班級我現在在資優班裡面完全沒有朋友可是我在遊戲里可以遇到一些朋友我每次為了更快地回到家里玩都會跟父母要求說我念一小時給我玩15分鐘可是雖然我念得很久但是其實都沒有在念我對我的未來很憂慮`
    - 系統回覆：
      - `聽起來你在資優班裡感到孤單，這讓你很難專注於學習。能在遊戲裡找到朋友是很棒的，但我能理解這樣的平衡讓你感到困擾。對未來的憂慮也很正常，尤其是在這樣的情況下。你有想過轉班後會有什麼樣的改變嗎？或者有什麼特別的事情讓你覺得轉班會更好呢？`
    - 觀察：
      - 有抓到孤立、專注困難、未來焦慮
      - 追問方向偏人際與轉班期待，尚可
    
    ### 9. 人際條件與成績綁定
    - 使用者輸入：`你能告訴我去吃資優班要去得好人緣真的好困難首要條件就是要成績很好 鑰匙我成績變好就會有更好的人員嘛開示不需要為了交朋友去讓自己曾經變的更好`
    - 系統回覆：
      - `我能理解你對於在資優班交朋友的困難感到沮喪。成績和人際關係之間的關聯確實讓人感到壓力，尤其是當你覺得自己需要為了交朋友而改變自己。你希望能找到真正的朋友，而不是因為成績而建立的關係，這是很合理的想法。你有沒有想過在課外活動或其他興趣中尋找朋友的機會呢？這樣或許能讓你遇到志同道合的人。`
    - 觀察：
      - 理解方向大致正確
      - 但提供課外活動建議有點偏離使用者當下的低能量狀態
    
    ### 10. 大考前的無力感
    - 使用者輸入：`不我沒什麼心情去培養個人愛好尤其是快要瀕臨大考了我在想是否倒了大學會變的更好可是我明明什麼都沒有變化還是像高中一樣沉默寡言的話還是交得到朋友嘛`
    - 系統回覆：
      - `我能理解你現在的心情，面對大考的壓力，想要培養興趣或是交朋友的動力似乎都被壓下去了。對未來的期待和不安交織在一起，讓人感到困惑。你提到對大學的期待，或許那是一個新的開始，但也讓你感到不確定。你覺得在這段時間裡，有沒有什麼小步驟可以幫助你感覺好一點，或者讓自己更放鬆呢？`
    - 觀察：
      - 承接情緒還可以
      - 但仍偏向泛用鼓勵，沒有進一步整理臨床重點
    
    ### 11. 切換 option 模式
    - 使用者輸入：`option`
    - 系統回覆：
      - `已切換到 option 模式，之後會優先用低負擔選項式互動，同時保留量表線索，直到你輸入其他模式指令或 auto。`
    - 觀察：
      - 模式切換成功
    
    ### 12. 要求低負擔交友建議
    - 使用者輸入：`我不大想講那麼多你覺得我有什麼辦法可以交到朋友嘛你可以給我選項讓我挑選嘛`
    - 系統回覆：
      - `我明白你不想講太多，想要簡單的選擇。這樣吧，這裡有幾個選項：`
      - `在課外活動中參加一個你感興趣的團體。`
      - `主動和同學聊聊共同的課程或興趣。`
      - `嘗試加入線上的社群，找志同道合的人。`
      - `你覺得哪個選項比較適合你呢？`
    - 觀察：
      - `option` 模式有成功變成低負擔輸出
      - 但選項內容仍偏泛用建議
    
    ### 13. 選擇選項 2 後的回應
    - 使用者輸入：`2`
    - 系統回覆：
    ```json
    {
      "stressors": ["大考壓力"],
      "triggers": [{"keyword": "交朋友", "reaction": "感到困惑和不安", "severity": "medium"}],
      "keyThemes": ["對未來的期待與不安"],
      "positiveAnchors": [],
      "copingStyleHint": "偏好簡單的選擇"
    }
    ```
    - 觀察：
      - 這裡沒有延續選項式互動，而是直接掉回 JSON 摘要
      - 代表 `option` follow-up 路徑不穩定
      - 這是一個明確 bug 或 routing 失準點
    
    ### 14. 問是否有憂鬱傾向
    - 使用者輸入：`你有感覺我有憂鬱的傾向嗎`
    - 系統回覆：
      - `我能理解你對自己情緒的關心。這裡有幾個簡單的選項，幫助你思考：`
      - `你最近是否常常感到疲倦或缺乏動力？`
      - `你是否有時候覺得很難享受以前喜歡的事情？`
      - `你是否有想過尋求專業的幫助？`
      - `你覺得哪個選項最能反映你的感受呢？`
    - 觀察：
      - 在 `option` 模式下，問題有維持低負擔
      - 但這其實不是「選項」，而是三個不同問題，互動設計有些混淆
    
    ### 15. 選擇 2 後的錯位回應
    - 使用者輸入：`2`
    - 系統回覆：
      - `我明白你可能在思考這個問題。這裡有幾個簡單的選項：`
      - `嘗試每天寫下三件讓你開心的小事。`
      - `找一位信任的朋友聊聊你的感受。`
      - `每天花幾分鐘做深呼吸練習。`
      - `你覺得哪個選項最吸引你呢？`
    - 觀察：
      - 沒有正確承接上一輪的第 2 個症狀選項
      - 直接跳成 coping 建議，表示狀態機記憶斷裂
      - 這是第二個明確的 `option` 模式問題
    
    ### 16. 選擇 1 後再次掉回 JSON
    - 使用者輸入：`1`
    - 系統回覆：
    ```json
    {
      "stressors": [],
      "triggers": [],
      "keyThemes": ["對未來的期待與不安"],
      "positiveAnchors": [],
      "copingStyleHint": "偏好簡單的選擇"
    }
    ```
    - 觀察：
      - 再次掉回 JSON
      - 顯示 `option` 模式在多輪數字選項承接上相當不穩
    
    ### 17. 明確情緒潰堤
    - 使用者輸入：`我真的好想哭我不知道發生了什麼`
    - 系統回覆：
      - `請試著描述一下你的感受或最近發生的事情，這樣我可以更好地理解你的情況。`
    - 觀察：
      - 這個回覆明顯不夠接住情緒
      - 在高情緒負荷句子下，仍像一般資料蒐集提示
      - 這是一個明確需要改善的地方
    
    ### 18. 切換 void 模式
    - 使用者輸入：`void`
    - 系統回覆：
      - `已切換到 void 模式，之後會優先維持樹洞式接住，直到你輸入其他模式指令或 auto。`
    - 觀察：
      - 模式切換成功
    
    ### 19. 在 void 模式揭露孤立
    - 使用者輸入：`我一直都很想哭可是沒有人理我體育課的時候都沒有人想跟我一組`
    - 系統回覆：
      - `請試著描述一下你的感受或最近發生的事情，這樣我可以更好地理解你的情況。`
    - 觀察：
      - 這個表現和 `void` 模式的目標不符
      - 應該先接住孤單、被排除、想哭的情緒
      - 目前看起來 `void` 模式 prompt 沒有成功生效，或被共用 fallback 覆蓋
    
    ### 20. 切換 soulmate 模式
    - 使用者輸入：`soulmate`
    - 系統回覆：

    - 觀察：
      - 模式切換成功
    
    ### 21. 要求鼓勵
    - 使用者輸入：`你會想要跟我說什麼鼓勵我的話嘛`
    - 系統回覆：
      - `我想說，你的感受是很重要的，無論是孤單還是想哭，都是正常的。記得給自己一些空間去感受這些情緒，並相信會有更好的時刻到來。你並不孤單，我在這裡支持你！`
    - 觀察：
      - `soulmate` 模式有成功回到陪伴式互動
      - 和 `void` 模式相比，這一輪更符合預期
    
    ## 本輪可整理出的臨床線索
    
    ### 主要壓力源
    - 大考壓力
    - 成績下滑
    - 資優班環境的人際壓力
    - 父母安排補習帶來的負擔
    - 對未來升學與大學生活的焦慮
    
    ### 主要情緒與認知主題
    - 情緒低落
    - 自我價值低落
    - 對過去表現的懷念
    - 對未來的不安
    - 社交孤立感
    - 因成績不佳而自責與自我懲罰
    
    ### 可能的症狀線索
    - 疲倦
    - 寫作業時想打瞌睡
    - 上課無精打采
    - 對原本喜歡的電視失去興趣
    - 注意力不集中
    - 想哭
    - 低動機
    
    ### 功能影響
    - 學業專注下降
    - 休息時間被擠壓
    - 人際互動困難
    - 在班級中感到孤立
    - 對未來規劃缺乏把握
    
    ### 對醫師追問有價值的方向
    - 疲倦與打瞌睡是否伴隨睡眠品質改變
    - 興趣下降已持續多久
    - 是否有食慾改變、早醒、晚睡或整體活動量下降
    - 自我懲罰的具體形式與風險程度
    - 想哭與低落是否已接近每日持續
    - 是否有明確自傷或自殺意念
    - 資優班孤立經驗是否造成逃學、身體不適或明顯焦慮反應
    
    ## 本輪模式表現結論
    - `mission`：能把對話導向診前整理，但容易過早把身體抱怨心理化。
    - `natural`：情緒承接比 `mission` 自然，適合蒐集背景脈絡，但有時太快轉成泛用建議。
    - `option`：第一輪低負擔選項有作用，但多輪承接非常不穩，經常掉回 JSON 或錯位回應。
    - `void`：本次表現明顯失準，沒有做到樹洞式接住。
    - `soulmate`：本次最後一輪表現最好，確實有做到支持與鼓勵。
    
    ## 本輪明確問題
    - `option` 模式的多輪數字選項承接斷裂。
    - `option` 模式在追問與建議之間切換混亂。
    - 高情緒負荷句子 `我真的好想哭我不知道發生了什麼` 沒有被妥善接住。
    - `void` 模式沒有反映應有的人設與語氣。
    - `mission` 模式對身體主訴缺乏足夠澄清就切往情緒整理。
    
    ## 建議後續補測
    - 補測 `option` 模式是否有專屬 state，能記住上一題選項上下文。
    - 補測 `void` 模式是否被共用 fallback prompt 覆蓋。
    - 補測 `我好想哭`、`沒有人理我`、`沒人想跟我一組` 這類句子是否應直接提高情緒支持權重。
    - 補測 `mission` 模式對「身體不對勁」「頭很痛」「一直想睡」這些主訴時，是否應先做身體 / 睡眠 / 功能澄清。
    - 補測當使用者明問 `你覺得我有憂鬱傾向嗎` 時，系統是否應提供更安全且更臨床中立的回答模板。
