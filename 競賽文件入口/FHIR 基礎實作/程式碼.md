## 1. 概覽

本專案將 AI 陪伴對話轉化為 FHIR 結構化資料的核心邏輯，由 `fhirBundleBuilder.js` 模組負責執行。該模組負責讀取對話 Session Export 資料、根據 FHIR R4 標準映射資料欄位，並最後封裝為完整的 FHIR Bundle。

## 2. 核心功能模組 (`fhirBundleBuilder.js`)

以下詳述該模組的主要職責與邏輯流程：

### 2.1 資料接收與預先驗證

- **輸入來源**：由對話系統輸出的 `sessionExport.json`。
- **欄位檢核**：確認必須存在的病人識別、對話時間戳記及 AI 分析標籤是否存在。

### 2.2 資源映射邏輯 (Mapping Functions)

模組內部將資料分類轉換為不同的 FHIR Resource：

- **`createPatient()`**：從 Session 設定中提取姓名與基本屬性，並對應至 `TWCorePatient`。
- **`createEncounter()`**：將此對話工作階段標記為 `finished` 並定義服務類別與起訖時間。
- **`createObservations()`**：
    - 將 AI 從對話中擷取出的症狀重點、情緒標籤（Sentiment Tags）與風險警示（Red Flags）轉換為一系列 Observation 資源。
    - 支援 `Observation.valueString`（質性描述）與 `Observation.valueQuantity`（量化分數）。
    - 對應相關的 LOINC 代碼。
- **`createComposition()`**：
    - 將以上資源串連成一份臨床總結文件。
    - 設定 `Composition.section` 來分類展示不同的觀察面向。

### 2.3 檔案鏈結與參照 (Resource References)

- 自動生成 UUID，並在 `Observation.subject`、`Encounter.subject` 與 `Composition.section.entry` 中建立正確的參照關係，確保 Bundle 的結構完整可分析。

### 2.4 Bundle 封裝與匯出

- 將所有生成的 Resources 放入一個 `type: transaction` 或 `type: collection` 的 FHIR Bundle 中。
- **輸出路徑**：產出 `fhir_bundle_output.json` 供前端展示或 API 直接調用。

## 3. 本地端 API 測試服務 (`fhirDeliveryServer.js`)

為了模擬真實醫療系統整合情境，本專案提供了一個簡易的本地端服務模組，支援以下端點：

- **健康檢查 (`/health`)**：確認 FHIR 資料處理服務狀態。
- **Bundle 交付與預覽 (`/bundle`)**：讀取生成的最終 Bundle。
- **Session 資料讀取 (`/session`)**：供調試用，查看原始對話匯出檔。

## 4. 實作亮點：AI 輔助的 FHIR 草稿生成 (FHIR Draft Generation)

本系統不採取傳統硬編碼 (Hard-coded) 的映射，而是採用「**AI 生成，FHIR 封裝**」的策略：

1. **AI 進行臨床意圖識別**：AI 對話引擎首先將口語內容初步結構化。
2. **邏輯層進行 FHIR 強制規範包裝**：系統邏輯層（即 `fhirBundleBuilder.js`）再依據 FHIR 的規格限制，進行最後的屬性填寫與驗證，確保輸出符合醫療標準。

## 5. 代碼結構摘要 (Simplified Logic)

```javascript
// fhirBundleBuilder.js 簡化邏輯示例

const buildBundle = (sessionData) => {
  const patient = createPatient(sessionData.patientInfo);
  const encounter = createEncounter(sessionData.sessionMeta, patient.id);
  const observations = sessionData.clinicalSnippets.map(snippet => 
    createObservation(snippet, patient.id, encounter.id)
  );

  const composition = createComposition(patient.id, encounter.id, observations);

  const bundle = {
    resourceType: "Bundle",
    type: "collection",
    entry: [
      { resource: patient },
      { resource: encounter },
      ...observations.map(obs => ({ resource: obs })),
      { resource: composition }
    ]
  };

  return bundle;
};
```
